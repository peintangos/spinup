# セキュリティベースライン

AWS AIDLC Security Baseline に基づく、コードレベルで確認すべきセキュリティ観点。

## チェック項目

### 暗号化（SEC-01）
- 外部通信はすべて HTTPS/TLS 1.2+ を使用しているか
- 平文プロトコル（HTTP, FTP 等）を使用していないか

### ロギング（SEC-03）
- ログにパスワード、トークン、API キー、PII を出力していないか
- エラーログにスタックトレースや内部情報を過剰に含めていないか

### セキュリティヘッダー（SEC-04）
- Web アプリケーションの場合、CSP / HSTS / X-Content-Type-Options 等を設定しているか

### 入力バリデーション（SEC-05）
- 外部入力に対して型チェック・長さ制限・フォーマット検証を行っているか
- SQL クエリにパラメータ化クエリを使用しているか（文字列結合による組み立て禁止）
- HTML/スクリプトのエスケープによる XSS 対策がされているか

### 最小権限（SEC-06）
- ワイルドカード権限を使用していないか
- 読み取り/書き込み権限が分離されているか

### アクセス制御（SEC-08）
- すべてのルートに認証が設定されているか（明示的に公開とするもの以外）
- CORS がワイルドカードになっていないか（認証付きエンドポイント）
- JWT/トークンの検証がリクエストごとに行われているか

### 設定ミス防止（SEC-09）
- デフォルト資格情報を使用していないか
- エラーメッセージにスタックトレースや内部詳細を含めていないか
- クラウドストレージのパブリックアクセスがブロックされているか

### サプライチェーン（SEC-10）
- lock ファイル（package-lock.json）がコミットされているか
- 未使用の依存関係がないか
- Docker/CI で `latest` タグを使用していないか

### 資格情報管理（SEC-12）
- ハードコードされた資格情報（API キー、パスワード等）がないか
- シークレットは環境変数またはシークレットマネージャーで管理されているか
- `.env` ファイルが `.gitignore` に含まれているか

### 例外処理（SEC-15）
- 外部呼び出しに明示的なエラーハンドリングがあるか
- エラー時にアクセスを拒否する（フェイルクローズ）設計か
- エラーパスでリソースが適切にクリーンアップされるか
- グローバルエラーハンドラが設定されているか
